System design

Technology & Architecture

SwachhSense is designed as a full IoT pipeline. In this academic prototype the hardware layer is simulated in the browser — but the data model, thresholds and workflows are exactly what a physical deployment would use.

Data pipeline

Layer 1 · Edge

Sensor node

ESP32 microcontroller with MQ-137 ammonia sensor, HC-SR04 ultrasonic level sensors (water & soap), IR footfall counter and per-stall occupancy switches.

Layer 2 · Transport

Telemetry

Readings are packaged as JSON packets every 3 seconds and streamed over Wi-Fi. The dashboard's telemetry console shows this raw stream.

Layer 3 · Logic

Rule engine

Each packet is scored against hygiene thresholds. Breaches raise alerts with severity, and cleanliness is computed as a rolling 0–100 score.

Layer 4 · Presentation

Command centre

Live KPIs, a city map, facility drill-downs, alert acknowledgement, crew dispatch, citizen reports and CSV export for maintenance logs.

Software stack

Frontend

HTML5, CSS3 and vanilla JavaScript. Charts rendered with Chart.js 4; the city map is hand-built SVG.

Authentication

Firebase Authentication (email/password). A route guard redirects signed-out visitors to the login page before any protected content renders.

Hosting & delivery

Deployed on Firebase Hosting with a global CDN. Source is version-controlled on GitHub with feature-based commits from every team member.

Simulation engine

A software model of six ESP32 nodes generates realistic telemetry — daily footfall curves, ammonia build-up between cleanings, supply depletion and random incidents.

Rule engine

Threshold evaluation, alert lifecycle (raise → acknowledge → resolve), refill ETAs and a composite cleanliness score per facility.

Planned hardware

ESP32-WROOM-32, MQ-137, HC-SR04 ×2, IR break-beam counter, reed switches per stall — roughly ₹2,400 per facility at pilot scale.
